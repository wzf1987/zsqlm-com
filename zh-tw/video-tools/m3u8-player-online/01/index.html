你在 iPhone 刷 YouTube、用 Netflix 看劇、上 線上課程，背後都有同一個技術在默默運作——M3U8 協議。它是當今串流媒體時代的「隱形骨架」，卻很少有人真正了解它。今天，我們就來徹底拆解這個技術。

M3U8 到底是什麼？
#

M3U8，全稱 M3U（MP3 URL）的 UTF-8 編碼版本，本質上是一個純文字格式的播放列表索引檔案。它並不儲存影片本身，而是像一張「導航地圖」，告訴播放器：影片被切成了哪些小塊、每一塊在哪裡、該按什麼順序播放。

M3U8 是 HLS（HTTP Live Streaming）協議的核心組成部分，由 Apple（蘋果公司） 於 2015 年前後推廣。如今，它已取代過去的 Real 格式播放列表，成為 iOS、Android、Windows、macOS 乃至 智慧電視的串流媒體標準。

簡單一句話：M3U8 不是影片，而是影片分片的「目錄清單」。

M3U8 的工作原理：切片＋索引＋自適應碼率
#

M3U8 之所以能統治串流媒體江湖，靠的是三板斧：

1. 媒體切片
#

原始影片（如 MP4）被切割為若干秒一個的 TS 分片（通常 2–10 秒）。每個分片獨立可下載、獨立可播放，天然支援斷點續傳。

2. 索引檔案管理
#

FFmpeg 等工具根據 TS 分片生成 M3U8 檔案，透過 Nginx、Apache 等普通 Web 伺服器分發。播放器先解析 M3U8，再按順序拉取分片並拼接播放。

3. 自適應碼率（ABR）——最強殺手鐧
#

一個 Master Playlist（主 M3U8） 可以指向多個不同畫質的子串流：

#EXT-X-STREAM-INF:BANDWIDTH=1280000,RESOLUTION=640x360 low.m3u8
#EXT-X-STREAM-INF:BANDWIDTH=2560000,RESOLUTION=1280x720 mid.m3u8
#EXT-X-STREAM-INF:BANDWIDTH=5120000,RESOLUTION=1920x1080 high.m3u8

網路差時自動降畫質，網路好時無縫升畫質，用戶幾乎無感知。

M3U8 檔案結構：打開就能看懂
#

用記事本打開一個 M3U8 檔案，你會看到這樣的內容：

標籤作用
#EXTM3U檔案標識，必須放首行
#EXT-X-VERSION協議版本號，決定兼容特性
#EXT-X-TARGETDURATION單個分片最大時長（秒）
#EXTINF分片時長＋URL，核心數據
#EXT-X-KEY加密資訊（常見 AES-128）
#EXT-X-ENDLIST點播結束標識（直播無此標籤）

直播場景下，伺服器會持續追加新分片、淘汰舊分片，M3U8 檔案即時更新，實現低延遲直播。

M3U8 的優勢與局限
#

優勢非常明顯：

  • ✅ 跨平台無敵：幾乎所有現代設備原生支援
  • ✅ 部署極簡：普通 HTTP 伺服器即可，無需 RTMP 等專用服務
  • ✅ 容錯性高：單個分片失敗不影響整體，可重試或跳過
  • ✅ 直播友好：即時更新列表，賽事、網課、秀場直播的首選

局限也要正視：

  • ⚠️ 直播延遲通常 3–30 秒，低於 WebRTC 等低延遲協議
  • ⚠️ 長影片產生大量 TS 分片，檔案管理成本高
  • ⚠️ 商用場景多帶 AES-128 加密，普通工具無法直接下載
